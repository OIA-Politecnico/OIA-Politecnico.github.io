## A - Feed the Beast

Notar que el predicado `P(x) = es posible hacerlo por x dias` cumple las
condiciones para hacer busqueda binaria. Entonces hacemos busqueda binaria sobre
la respuesta para encontrar la maxima cantidad de dias que es posible completar.

Una vez fijada la cantidad de dias `m`, es facil calcular la cantidad de cajas
para la `i`-esima casa: es `m * a[i] / x`, redondeando para arriba.

```c++
#include <bits/stdc++.h>
using namespace std;

#define forr(i,a,b) for(int i = int(a); i < int(b); ++i)
#define forn(i,n) forr(i,0,n)
#define all(x) begin(x),end(x)

using ll = long long;

// division redondeada hacia arriba
__int128 cd(__int128 a, __int128 b) { return (a+b-1) / b; }

int main() {
    int n, b, x;
    cin >> n >> b >> x;
    
    vector<int> a(n);
    forn(i, n) cin >> a[i];
    
    ll daily = accumulate(all(a), 0ll);
    ll lo = 0;                     // sin duda es posible 0 dias
    ll hi = ll(b) * x / daily + 1; // sin duda es imposible. Tambien funcionaria poner 1000000000001 (13 digitos)
    
    while (hi - lo > 1) {
        ll mi = (lo + hi) / 2;
        
        __int128 need = 0;
        forn(i, n) need += cd(a[i] * mi, x);
        if (need <= b) lo = mi;
        else hi = mi;
    }
    
    cout << lo << "\n";
}
```

> En esta solucion se usa `__int128`, pero es posible evitarlo si tenemos un
> poco mas de cuidado con las cuentas.

## B - Phoenix and Puzzle

Podemos usar 2 triangulos para armar un cuadrado de 1x1. Si queremos hacer un
cuadrado de NxN, usamos N^2 cuadrados, o 2xN^2 triangulos.

Como muestra la figura del enunciado, tambien podemos usar 4 triangulos para
hacer un super cuadrado de 1x1. Si queremos hacer uno de NxN, usamos 4xN^2
triangulos.

Como la maxima cantidad de triangulos no es tan alta, podemos construir todas
las cantidades validas, meterlas en un arreglo ordenado, y despues usar
`std::binary_search()` para chequear si una cantidad dada en el input aparece en
ese arreglo.

```c++
#include <bits/stdc++.h>
using namespace std;

#define forr(i,a,b) for(int i = int(a); i < int(b); ++i)
#define forn(i,n) forr(i,0,n)
#define pb push_back
#define all(x) begin(x),end(x)

int main() {
    
    vector<int> a;
    forr(i, 1, 23000) {
        a.pb(i*i*2);
        a.pb(i*i*4);
    }
    sort(all(a));
    
    int n; cin >> n;
    
    forn(_, n) {
        int k; cin >> k;
        if (binary_search(all(a), k)) cout << "YES\n";
        else cout << "NO\n";
    }
}
```

		hi--;

		};
	};

		cout << R[i] << " ";
	};

## F - Santa Claus and Keyboard Check

Hacemos una primera pasada registrando los pares de teclas intercambiadas. En
particular lo anotamos en una tabla que asocia cada letra con la que esta
intercambiada.

Es posible que la misma tecla aparezca varias veces intercambiada con distintas
teclas. Para detectar esto, hacemos otra pasada por las cadenas viendo si en
alguna posicion hay un intercambio distinto al registrado en la tabla.

> Se pueden hacer las dos cosas en una sola pasada pero me parecio mas facil
> codearlo asi

Despues iteramos por todas las letras y nos guardamos un par de caracteres por
cada letra que no esta asociada con si misma. Aparte, para no registrar cada par
dos veces (una por cada letra que lo compone), solo guardamos el par al pasar
por la letra menor de las dos (o sea, solo si la letra registrada en la tabla es
mayor)

```c++
#include <bits/stdc++.h>
using namespace std;

#define forr(i,a,b) for(int i = int(a); i < int(b); ++i)
#define forn(i,n) forr(i,0,n)
#define pb push_back
#define all(x) begin(x),end(x)
#define sz(x) (int((x).size()))

int main() {
    string s, t;
    cin >> s >> t;
    
    int n = sz(s);
    
    char r[256];
    fill(all(r), 0);
    
    forn(i, n) {
        r[s[i]] = t[i];
        r[t[i]] = s[i];
    }

    forn(i, n) {
        if (r[s[i]] != t[i] || r[t[i]] != s[i]) {
            cout << "-1\n";
            return 0;
        }
    }
    
    vector<pair<char, char>> ps;
    forn(i, 256) {
        if (i < r[i]) {
            ps.pb({(char)i, r[i]});
        }
    }
    
    cout << sz(ps) << "\n";
    for (auto [x, y] : ps) {
        cout << x << " " << y << "\n";
    }
}
```

## G - Increase Subarray Sums

> **Observación**: Al sumar `x` a *cualquier* posición de un subarreglo, su suma total aumenta en `x`.

Es decir, da igual sumárselos "a los negativos" o a los positivos, siempre que todos estén en el mismo subarreglo.

Entonces, si el subarreglo que tomamos para calcular *f(k)* fuera de longitud *l*, tomaríamos el subarreglo de *mayor suma* entre todos los subarreglos de dicha longitud. Así, es el que mayor suma tendría tras aumentar *k* de sus elementos en *x* unidades (sumando `k*x` unidades). Observar que como las posiciones que elegimos deben ser todas distintas, *l* debe ser mayor o igual a *k*.

Por lo tanto, como `n <= 5000`, se pueden iterar por todos los subarreglos computando su suma, y almacenar la mayor suma obtenida entre todos los subarreglos de cada longitud. Con esa información, se puede obtener con otra pasada el subarreglo de mayor suma de longitud **mayor o igual** a cada *l*.

Finalmente, la respuesta de *f(k)* será bien el máximo entre aumentar la suma total del subarreglo de longitud *<= k* de *mayor suma* en `k*x` unidades (tomando *k* posiciones dentro y sumándole a c/u *x*), y el mayor valor de *f(k')* para *k'* < *k*. Esto es así ya que podría suceder que todos los subarreglos de longitud mayor o igual que *k* tengan una suma menor a subarreglos de menor longitud (por ejemplo, si obliga a tomar números negativos que con longitudes más cortas se pueden evitar). Entonces el mejor resultado de *f(k)* sería hacer lo mismo que para *f(k')*, y luego usar las operaciones *k-k'* restantes en aumentar otras posiciones cualesquiera fuera del subarreglo usado en *f(k')*. Es por esto que se aprecia en el ejemplo que la lista de resultados es *no decreciente*.

```c++
#include<bits/stdc++.h>
 
#define forn(i,n) for(int i = 0; i < int(n); i++)
#define forsn(i,s,n) for(int i = int(s); i < int(n); i++)
#define dforn(i,n) for(int i = int(n)-1; i >= 0; i--)
#define dforsn(i,s,n) for(int i = int(n)-1; i >= int(s); i--)
 
using namespace std;
typedef long long ll;

int const MAXN = 5005;
int const INF = 2e9;

int arr[MAXN],auxi[MAXN];

int main() {

    int t; cin >> t;
    forn(_,t) {
        int n,x; cin >> n >> x;
        forn(i,n) cin >> arr[i];
        forn(i,n+1) auxi[i] = -INF;

        forn(i,n) {
            int s = 0;
            forsn(j,i,n) {
                s += arr[j];
                auxi[j-i+1] = max(auxi[j-i+1], s);
            }
        }

        dforn(i,n) auxi[i] = max(auxi[i],auxi[i+1]);

        ll r = 0;
        forn(i,n+1) {
            r = max(r, auxi[i] + (ll)x*i);
            cout << r << ' ';
        }
        cout << '\n';
    }

    return 0;
}
```

## H - In Love

El problema nos pide determinar si existe un par de segmentos que no se toquen. El enunciado define que dos segmentos se tocan si alguno de sus extremos se encuentra dentro del otro segmento. Entonces, dados dos segmentos `[l,r]` y `[a,b]` **no** se tocan en dos casos:

* `l <= r < a <= b`:
```
            |-----|
|--------|
```

* `a <= b < l <= r`:
```
|--------|
            |-----|
```

Es decir, basta con que el extremo izquierdo de uno se encuentre a derecha del extremo derecho del otro. 

Entonces, consideremos el segmento `[l,r]` cuyo extremo derecho se encuentra *más a la izquierda* en un instante dado, y el segmento `[a,b]` cuyo extremo izquierdo se encuentra *más a la derecha* en un instante dado. O sea, los segmentos que "terminan antes o igual que el resto" y "comienzan igual o después que el resto". Si `r < a`, entonces los segmentos no se intersecan, y la respuesta es `YES`. Caso contrario, **no pueden existir** dos segmentos que no intersequen. 

En efecto, si existiera un par de segmentos tal, tiene que cumplirse que el extremo derecho del más a la izquierda sea estrictamente menor al extremo izquierdo del segmento más a la derecha del par. Sin embargo, eso no puede suceder porque eso contradeciría que `[l,r]` es el segmento tal que `r` es mínimo entre todos los segmentos, y que `[a,b]` es el segmento tal que `a` es máximo entre todos los segmentos, para los cuales dijimos que *r ≮ a*.

Por lo tanto, basta con guardar la lista de los extremos izquierdos y derechos de los segmentos en un par de `multiset`s, y determinar si el mínimo extremo derecho es menor al máximo extremo izquierdo o no.

> Es importante usar `multiset` porque varios segmentos pueden compartir alguno de sus extremos, y cuando se elimina alguno sólo se debe eliminar una de las apariciones de sus extremos en cada `multiset`. Recordar que `.erase()` sin un iterador elimina _todas_ las apariciones, por lo que lo usamos en combinación con `.find()` que devuelve un iterador que apunta a solo una.

```c++
#include<bits/stdc++.h>
#define forn(i,n) for(int i = 0; i < int(n); i++)
using namespace std;
multiset<int> ls,rs;
int main() {

    int q; cin >> q;
    forn(i,q) {
        char typ; cin >> typ;
        int l,r; cin >> l >> r;
        if (typ == '+') {
            ls.insert(l);
            rs.insert(r);
        }
        else {
            ls.erase(ls.find(l));
            rs.erase(rs.find(r));
        }

        if (!ls.empty() && (*rs.begin()) < (*ls.rbegin()))
            cout << "YES\n";
        else cout << "NO\n";
    }

    return 0;
}
```

## I - Lucky Number 2

Primero analizaremos cómo obtener una respuesta al problema, y luego determinaremos cómo obtener la respuesta **mínima** como pide el enunciado.

> **Observación 1**: La diferencia entre *a_3* y *a_4* **no** puede ser más que 1.

Si la diferencia es 0 (son iguales), independientemente de los valores de *a_1* y *a_2*, la respuesta podría ser `474747...474` o `747474...747`. Por ejemplo para *a_3 = a_4 = 3*:

```
4747474 = 47 47 47 4 = 4 74 74 74
```
```
7474747 = 7 47 47 47 = 74 74 74 7
```

En cambio, si la diferencia es 1, se puede eliminar el último dígito en cada caso. Si *a_3 = a_4+1*:
```
474747 = 47 47 47 = 4 74 74 7
```
Y si *a_4 = a_3+1*:
```
747474 = 74 74 74 = 7 47 47 4
```

¿Y si la diferencia es mayor que 1? Supongamos que tenemos *a_3 = 1* y *a_4 = 3*. Entonces nuestra respuesta sería de la forma:
```
... 74 ... 74 ... 74 ...
```
Si llenamos todos los espacios sin nada, ya tendríamos al menos dos `47`s:
```
747474 = 7 47 47 4
```
Entonces habría que reducir la cantidad de `47`s resultantes, insertando caracteres entre medio de algún par de `74`s. Sin embargo, sólo pueden meterse `4`s o `7`s, ya que sino dejaría de ser un lucky number. En cualquier caso, no se puede evitar tener otro `47`:
```
74 *4* 7474 = 74 47 47 4
```
```
74 *7* 7474 = 7 47 7 47 4
```
Análogo sería el razonamiento si *a_3 = 3* y *a_1 = 3*. En general, si la diferencia entre *a_3* y *a_4* es mayor a 1, se **debe devolver `-1`**.

> **Observación 2**: Si sobran `4`s o `7`s, se pueden insertar en la respuesta de antes al lado de otra aparición del mismo dígito sin cambiar *a_3* ni *a_4*.

Esta observación la vimos un poco antes, que sin importar dónde pongamos un `4` o un `7`, no podemos reducir la cantidad de `47`s o `74s`. Entonces, si antes llegamos a la siguiente respuesta haciendo cumplir los valores de *a_3* y *a_4*:
```
7474747
```
Y *a_1 = 5*, *a_2 = 8*, entonces sobran 2 `4`s y 3 `7`s. Así, una posible respuesta válida sería:
```
74 *44* 74 7 *777* 47 = 74447477747
```
O también:
```
7 *77* 474 *4* 7 *7* 4 *4* 7 = 777474477447
```

Observar también que la estrategia inicial de intercalar `4`s y `7`s permite cumplir con *a_3* y *a_4* utilizando la **mínima** cantidad de dígitos de cada tipo, ya que se intenta reutilizar para cada `47` o `74` un dígito del número anterior. Es decir, utilizamos la mínima cantidad posible de dígitos para cumplir con *a_3* y *a_4*. En caso de no alcanzar la cantidad de `4`s o `7`s con la estrategia anterior (*a_1* o *a_2* muy pequeños=, entonces deberemos en ese caso informar que no es posible obtener una respuesta a la entrada dada.

Un caso **borde** a tener en cuenta es cuando *a_3 = a_4*. En dicho caso, hay dos formas de obtener una respuesta: como `4747...474` o como `7474...747`. Observar que a pesar de contarse misma cantidad de `47`s y `74`s en ambos casos, el primero hace uso de un dígito `4` más que de `7`s, mientras en el segundo caso es al revés. Así, si nos dijeran que *a_3 = a_4 = 3* y que *a_1 = 3, a_2 = 4*, sólo la siguiente forma cumpliría con las condiciones:

```
7474747
```
Pues si hiciéramos:
```
4747474
```
Nos excederíamos de la cantidad de `4`s permitida. Si fuera en cambio *a_1 = 4, a_2 = 4*, no habría problema, y se agregaría el `7` que sobra donde sea oportuno:
```
47474774
```
En general, este caso se puede representar como que *a_1 = a_3 = a_4, a_2 = a_3 + 1 = a_4 + 1*

Con estas reglas, podemos generar una respuesta válida para el enunciado, con la excepción que no necesariamente es **mínima**. Para ello, debemos hacer algunas observaciones adicionales.

> **Observación 3**: Cuando la diferencia entre *a_3* y *a_4* es de exactamente 1 unidad, la forma de hacer cumplir la respuesta es una sola.

Esta observación surge de lo que vimos al principio de la solución. Si *a_3* y *a_4* son iguales, hay dos formas, a excepción del caso borde mencionado. Sin embargo, cuando la diferencia es de 1 (porque no puede ser mayor), hay una sóla forma de construir la solución al principio: alternando `4`s y `7`s, empezando por un `4` o un `7` si *a_3 > a_4* o *a_4 > a_3* respectivamente.

Por lo tanto, independientemente de la minimalidad de la respuesta, la forma de construir al principio la solución **debe** de ser de esta forma. Lo que hay que tener cuidado es *dónde* ponemos los `4`s y `7`s que sobren. La idea es hacerlo minimizando el número final. Entonces, todos los `4`s restantes deberán de ponerse junto con el **primer** `4` que aparezca en la respuesta construida, y todos los `7`s restantes junto con el **último** `7` que aparezca. De esta forma, se logra que los dígitos `4` (que tienen menor valor a los `7`) aparezcan en posiciones menores, y los dígitos `7` (que tienen mayor valor a los `4`) aparezcan en posiciones mayores, minimizando el resultado. En el código se puede ver cómo se logra esto en cada caso.

Falta entonces ver el caso cuando *a_3 = a_4*.

> **Observación 4**: Siempre que sea posible, elegir la forma `4747...474` cuando *a_3 = a_4*.

En efecto, si fueran posibles ambas formas de hacer cumplir *a_3 = a_4*, la forma `7474...747` ya no sería *mínima*, pues empieza con un `7`, que es un dígito de mayor valor que el `4` (y cualquier respuesta construida debe tener la misma cantidad de dígitos dadas las restricciones *a_1* y *a_2*). 

Decimos que "siempre que sea posible" pues, como vimos antes, hay un caso en el que la forma `7474...747` es la única posible: cuando *a_1 = a_3 = a_4, a_2 = a_3 + 1 = a_4 + 1*. Fuera de este caso, si *a_1 > a_3 = a_4*, se debe de optar por la otra forma.

Finalmente, en cada caso se deben de agregar los `4`s y `7`s restantes a fin de minimizar el resultado final, de forma similar a como lo describimos para el caso cuando la diferencia entre *a_3* y *a_4* es 1.

El código quedó algo desprolijo, pero tiene algunos comentarios que les pueden ayudar a guiarse.

```c++
#include<bits/stdc++.h>
 
#define forn(i,n) for(int i = 0; i < int(n); i++)
 
using namespace std;

int main() {
    int a,b,c,d; cin >> a >> b >> c >> d;

    if (abs(c-d) > 1) 
        return cout << "-1\n", 0;

    string s;

    if (c > d) {
        forn(i,c) s += "47";

        a -= c;
        b -= c;

        if (a < 0 || b < 0)
            return cout << "-1\n", 0;

        s = string(a,'4') + s;
        s += string(b,'7');

        cout << s << '\n';
    }
    else if (c < d) {
        forn(i,d) s += "74";

        b -= d;
        a -= d;

        if (a < 0 || b < 0)
            return cout << "-1\n", 0;
        
        s.pop_back();
        // cuando agrego 7s, al final de todo posible
        s += string(b,'7');
        s.pb('4');

        // cuando agrego 4s, al principio de todo posible
        s = "7" + string(a,'4') + s.substr(1);

        cout << s << '\n';

    }
    else if (c == d) {
        if (a > c) { // voy con la que me da un número más chico si me alcanza la cant de 4s
            forn(i,c) s += "47";

            a--; // saco el 4 de antemano

            a -= c;
            b -= c;

            if (a < 0 || b < 0)
                return cout << "-1\n", 0;

            s = string(a,'4') + s;
            s += string(b,'7');
            s.pb('4'); // pongo el 4 que saqué

            cout << s << '\n';
        }
        else { // sino a == c, en cuyo caso es de la forma 7474747 (menos 4s que 7s)
            forn(i,d) s += "74";

            b--; // saco el 7 de antemano

            b -= d;
            a -= d;

            if (a < 0 || b < 0)
                return cout << "-1\n", 0;
            
            s += string(a,'4');
            s += string(b+1,'7'); // +1 para contar el que saco de antemano, bien al final

            cout << s << '\n';
        }
    }

    return 0;
}
```
